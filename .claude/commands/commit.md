변경 사항을 확인하고 커밋을 생성하라.

## 규칙

1. `git status`와 `git diff`로 변경 내용을 확인한다
2. 현재 브랜치가 `main`이면 **커밋하지 않고 경고**한다 (작업 브랜치에서만 커밋)
3. 변경 내용을 분석하여 적절한 커밋 메시지를 작성한다
4. 커밋 메시지 형식: `{type}: {한국어 설명}`
   - `init:` - 초기 설정
   - `feat:` - 새 기능
   - `fix:` - 버그 수정
   - `refactor:` - 리팩토링
   - `chore:` - 설정/빌드 관련
   - `docs:` - 문서 수정
   - `test:` - 테스트 추가/수정
5. **Co-Authored-By 줄을 절대 포함하지 않는다**
6. CLAUDE.md, .claude/ 디렉토리는 .gitignore에 포함되어 있으므로 커밋하지 않는다
7. .env, credentials 등 민감한 파일은 커밋하지 않고 경고한다

## 추가 인자

- `$ARGUMENTS`가 있으면 해당 내용을 커밋 메시지에 반영한다
- 인자가 없으면 변경 내용을 분석하여 자동으로 메시지를 생성한다
