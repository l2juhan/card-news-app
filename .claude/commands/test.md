프로젝트의 테스트를 실행하고 결과를 보고하라.

필요에 따라 `/sc:test`를 활용하여 체계적으로 테스트를 수행한다.

## 실행 순서

1. **타입 체크**: `npx tsc --noEmit` 실행
2. **린트**: `npx eslint src/` 실행 (설정이 있는 경우)
3. **빌드 테스트**: `npm run build` 실행
4. **단위 테스트**: `npm test` 실행 (테스트가 있는 경우)
5. **E2E 테스트**: Playwright MCP가 사용 가능하면 활용

## SuperClaude 활용

| 상황 | 스킬 |
|------|------|
| 테스트 실행 및 커버리지 분석 | `/sc:test --coverage` |
| 테스트 실패 원인 분석 | `/sc:troubleshoot` |
| 테스트 코드 작성이 필요할 때 | `/sc:implement --with-tests` |
| 빌드 에러 디버깅 | `/sc:build --verbose` |

## 보고 형식

각 단계의 결과를 아래 형식으로 보고한다:

```
타입 체크:  ✓ 통과 / ✗ N개 에러
린트:      ✓ 통과 / ✗ N개 경고, M개 에러
빌드:      ✓ 성공 / ✗ 실패
테스트:    ✓ N개 통과 / ✗ N개 실패 / ⊘ 테스트 없음
```

## 에러 처리

- 에러가 발생하면 `/sc:troubleshoot`로 원인을 분석하고 수정 방법을 제안한다
- 수정이 간단한 경우 직접 수정 후 재테스트 (사용자 확인 후)
- 설정 파일이 아직 없는 항목(eslint, test 등)은 건너뛰고 `⊘ 미설정`으로 표시한다
