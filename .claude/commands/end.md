현재 Phase 작업을 마무리하고 문서 업데이트 + 커밋 + 푸시를 수행한다.

## 1. 현재 상태 확인

1. `git branch --show-current`로 현재 브랜치 확인
2. main 브랜치이면 **중단하고 경고** (작업 브랜치에서만 실행 가능)
3. `git status`로 커밋되지 않은 변경사항 확인
4. 커밋되지 않은 변경사항이 있으면 먼저 커밋 처리

## 2. 문서 업데이트

`/sc:document`를 활용하여 아래 문서들을 업데이트한다:

### PLAN.md
- 완료된 작업 항목을 `- [x]`로 체크
- 구현 중 변경된 사항이 있으면 반영 (기술 스택, 구조 등)

### CLAUDE.md
- 프로젝트 구조가 변경되었으면 디렉토리 트리 업데이트
- 새로운 개발 명령어가 추가되었으면 반영
- IPC 프로토콜이나 타입이 변경되었으면 반영
- 코딩 규칙에 추가할 내용이 있으면 반영

### README.md
- README.md가 없으면 새로 생성
- 있으면 현재 구현 상태에 맞게 업데이트
- 포함 내용: 프로젝트 소개, 설치 방법, 개발 명령어, 기술 스택

## 3. 최종 커밋 + 푸시

1. 문서 변경사항을 스테이징: `git add PLAN.md README.md` (CLAUDE.md는 gitignore)
2. 커밋 메시지: `docs: Phase {N} 완료 문서 업데이트`
3. **Co-Authored-By 줄을 절대 포함하지 않는다**
4. 리모트에 푸시: `git push origin {현재-브랜치}`
5. 푸시 완료 후 사용자에게 PR 생성 여부를 확인한다

## 4. PR 생성 (사용자 승인 시)

1. PR 제목: `Phase {N}: {Phase 목표 한줄 요약}`
2. PR 본문:
   ```
   ## Summary
   - 완료된 작업 목록 (PLAN.md 체크리스트 기반)

   ## Changes
   - 주요 변경 파일/모듈 요약

   ## Test plan
   - 검증 결과 (빌드, 타입체크 등)
   ```
3. base 브랜치: `main`
4. `gh pr create`로 생성 후 PR URL을 사용자에게 알려준다
